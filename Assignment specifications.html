Overview

This assignment uses three.js to implement a swim race in an olympic size pool. In part it builds on assignment 2.

In a basic assignment the swimmers will just glide up and down the pool with random speeds with no swim stroke action.

Next step up is to implement a simple swim stroke animation, where arms simply rotate about the shoulder.

Then next step up is to implement keyframe animation in Javascript, instead of using an inbuilt or library provided keyframe engine as in assignment 1, along with a keyframed swimming action. As shown in the images and video above.

The highest level assignment is to implement a medley swim race. A medley is a swim race of four different strokes: backstroke, breastroke, butterfly and freestyle. All four swim strokes are shown in the video below, and should be implemented using keyframe animation.

Details

Extend your program from assignment 2 to render a swim race with eight swimmers in a pool.

  • A swimming race with 8 swimmers in a 20m x 50m, 8 × 2.5m lane wide, 2.5m deep olympic pool. Random numbers are to be used to generated the speed of the swimmers, which may vary in a race. The swimmers do two lengths of the pool, as though in a 100m race. In the HD level assignment swimmers do a 200m medley which is 4 laps, each with a different stroke: butterfly, backstroke, breaststroke and freestyle.
  • Texture mapping of the pool and the swimmers.
  • Animating the swimmers in two ways: (1) simply moving them along the length of the pool (and back) with no swim stroke action (2) performing a swim stroke action using key frame animaton similar to the first assignment but with the keyframe animation implemented in Javascript.

Further Details

  • Use your hierarchical swimmer model from assignment 2. In case you did not get that done, use a block to represent the swimmers.
  • Include the same rendering controls: axes, lighting, wireframe.
  • The figure must be properly constructed according to the same hierarchy, to allow for easy animation. The figure must have a root node such as the body or a blank container node to allow the whole swimmer to be moved around the scene.
  • There should be a root scene object of which all components are children so that any transformations will transform the entire scene.
  • The swimmer should be able to be easily duplicated using a createSwimmer() function or something similar, such as the clone() method on the root node of the swimmer.
  • The race should start when the 'g' button is pressed, not before.
  • The swimmers will race from one end of the pool to the other, and then back. Each swimmer will have a random speed which can vary as the race proceeds. Use the speed to change the position of the swimmers using time deltas.
  • The swimmers color should be a shade of pink 0xff9966 ( or (255, 153, 102) or (1, 0.6, 0.4)).
  • A simple swim stroke animation approach is to have the arms constantly rotating around the shoulder.
  • A detailed swimmer stroke animation should be keyframed, as in assignment 1, with a keyframe animation engine implemented in Javascript using the interpolator discussed in the lectures and tutorials, and the functions: 
            function interpolate(keys, values, key)
        function lerp(k1, v1, k2, v2, k)
        function findInterval(keys, key)
  • Code should be simply annotated so that its easy to identify the different areas of your program. The assignment should be split into several logical files based on the content of the file.
